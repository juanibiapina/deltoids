deltoids adds tree-sitter scope to every diff — drop it into git as a pager,
or watch every edit your coding agent makes.

<h2 id="pager">Use as a diff pager</h2>

Drop-in for `git diff`, `git show`, lazygit, or any unified diff. Hunks expand
to show the enclosing function, class, or config block, so you don't have to
scroll to reconstruct context.

<Tabs>
  <TabItem label="Pipe">
    <Code code="git diff | deltoids" lang="bash" />
  </TabItem>
  <TabItem label="Git pager">
    <Code code="git config --global core.pager 'deltoids | less -R'" lang="bash" />
  </TabItem>
  <TabItem label="Lazygit">
    <Code code={`# ~/.config/lazygit/config.yml
git:
  paging:
    pager: deltoids`} lang="yaml" />
  </TabItem>
</Tabs>

<h2 id="agent">Follow your coding agent</h2>

Install the pi extension. Every file edit your agent makes flows through
deltoids — open `edit-tui` to watch the trace live, with full diffs and scope
context for each step.

<Code
  code={`pi install https://github.com/juanibiapina/deltoids
edit-tui`}
  lang="bash"
/>

*Under the hood, the extension swaps your agent's `edit` and `write` tools
for traced versions. Every call lands in a JSONL trace that `edit-tui`
browses.*

*Other agent integrations: coming.*

## Install

<Code
  code={`git clone https://github.com/juanibiapina/deltoids.git
cd deltoids

cargo install --path crates/deltoids-cli  # provides: deltoids
cargo install --path crates/edit-cli      # provides: edit, write, edit-tui`}
  lang="bash"
/>

The second line installs `edit` and `write` alongside `edit-tui`. They are
called by the pi integration — you don't invoke them directly.
